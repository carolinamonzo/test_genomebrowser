üîç Enhanced Names for Filtering

Transcript names contain embedded classification data in this format:

transcript_id|structural_category|associated_gene|subcategory|coding|FSM_class|RTS_stage

Example:

PB.118859.1|intergenic|novelGene_124|mono-exon|non_coding|A|False

This allows you to filter transcripts directly in the UCSC Genome Browser using any classification value!

üîç Advanced Filtering Instructions

This track supports advanced filtering directly in the UCSC Genome Browser, similar to the LRGASP hub!

How to Use Advanced Filters:

  1. Right-click on the track name in the UCSC Genome Browser
  2. Select "Configure" to access filter options
  3. Use the filter panels to select specific values
  4. Or use "Filter by name" for text-based searching

Available Filter Categories:

ALL 48 columns from the SQANTI3 classification file are now available for filtering!

  • Structural Category: Filter by transcript classification (intergenic, novelGene, novel_in_catalog, etc.)
  • Associated Gene: Filter by gene names or novel gene identifiers
  • Subcategory: Filter by exon structure (mono-exon, multi-exon, novel splice sites)
  • Coding Status: Filter by coding vs non-coding transcripts
  • FSM Class: Filter by Full Splice Match classification (A, B, C, D)
  • RTS Stage: Filter by RTS stage information
  • Plus 42+ additional columns: Including quality metrics, length statistics, splice site information, and more!

Filter Features:

  • Range-based filtering for numeric values
  • Text-based filtering with regex support
  • Multiple filter combinations for complex queries
  • Search indexing for fast transcript name lookups
