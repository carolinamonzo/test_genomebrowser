This track displays SQANTI3 transcriptome analysis results with embedded classification data for filtering.

üîç Enhanced Names for Filtering

Transcript names contain embedded classification data in this format:

transcript_id|structural_category|associated_gene|subcategory|coding|FSM_class|RTS_stage

Example:

PB.118859.1|intergenic|novelGene_124|mono-exon|non_coding|A|False

This allows you to filter transcripts directly in the UCSC Genome Browser using any classification value!

üîç Filtering Instructions

This track supports filtering directly in the UCSC Genome Browser using the complete filtering configuration!

Filtering Configuration:

  • filterValues: Defines the available filter values for each classification field
  • filterType: Set to "values" for discrete value filtering
  • filterBy: Specifies which fields can be used for filtering

How to Filter:

  1. Right-click on the track name in the UCSC Genome Browser
  2. Select "Filter by name" or "Filter by value"
  3. Enter search terms to filter transcripts
  4. The browser will use the configured filterBy fields for searching

Available Filter Values:

  • Structural Categories: intergenic, novelGene, novel_in_catalog, novel_not_in_catalog, antisense, sense_intronic, fusion, genic
  • Coding Status: coding, non_coding
  • FSM Classes: A, B, C, D
  • Subcategories: mono-exon, multi-exon, at_least_one_novel_splicesite

Example Filter Queries:

  • Type intergenic to show only intergenic transcripts
  • Type coding to show only coding transcripts
  • Type A to show only FSM class A transcripts
  • Type mono-exon to show only mono-exon transcripts

üí° Tip: The classification data is embedded in the transcript names, so you can search for any classification value directly in the UCSC filter interface!
